    • 面试问题

    1. Java 创建 String 有哪些方式，有什么区别？

    • 通过双引号直接赋值。 String str = "Hello world"
    • 通过构造器创建。 String str = new String("Hello world")

    双引号方式，存储在方法区的字符串池，相同的字面值无论被创建多少次，始终只有一个内存地址被分配
    构造器方式，存储在 heap 堆，每个对象分配一个内存地址

    2. String a = "abc" 和 String b = new String("abc")， a == b 的值是？

    False。存储的区域不同，内存地址不同，前者保存在方法区的字符串池，后者保存在 heap 堆

    3. String a = "abc" 存放在 java 虚拟内存的哪个区域？

    方法区的字符串池

    4. 字符串可以被修改吗？为什么做成这样？

    不可修改。设计成这样是为了安全和效率。

    • 安全：不可变类天生线程安全
    • 安全：字符串常用于网络连接、数据库连接时的参数，不可变可确保参数指向不被改变，确保安全
    • 效率：为了便于实现字符串池。字符串池可避免重复字符串，节省内存空间
    • 效率：不可变使得 String 对象在创建时，可以安心缓存 hashcode，再次使用时不用重新计算，从而提高效率。这也是 HashMap 经常使用 String 类型作为键值的原因

    5. StringBuilder 和 StringBuffer 线程安全吗？

    StringBuilder 线程不安全，StringBuffer 线程安全。除此之外，它们的原理和操作基本相同

    6. String, StringBuffer, StringBuilder 三个区别？

    • String 是不可变类，线程天生安全。每次修改都会引发新的 String 对象生成
    • StringBuffer 是可变类，线程不安全
    • StringBuilder 是可变类，线程安全。单线程中的性能比 StringBuffer 更高

    7. 和 StringBuilder 相比，StringBuffer 的线程安全是怎么处理的？

    通过 synchronized 关键字实现同步操作

    StringBuffer.append()

    StringBuilder.append()
