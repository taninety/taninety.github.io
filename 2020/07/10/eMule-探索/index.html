2020年了，我竟然重新对 emule 产生了兴趣
记录几个使用过程中遇到的问题

连接问题

1.服务器连接

“从 URL 更新 server.met” 输入以下任一网址，点击“更新”

  • http://upd.emule-security.org/server.met
  • http://gruk.org/server.met.gz

2.Kad 连接

“从以下网址获取节点信息” 输入以下网址，点击“引导程序”

  • http://upd.emule-security.org/nodes.dat
    或者手动下载上面的文件，放到相应目录

LowID 问题

解决这个问题有两个关键

1.确保自己的 IP 是公网 IP

确认方法：通过提供当前 IP 查询的网站确认自己连接到公网时的 IP，然后查询自己上网猫中网络侧信中息的 IP 地址，如果两个 IP 一致，说明有公网 IP
经过确认，发现自己是公网 IP，很幸运，可以进行下一步了

2.为 emule 端口设置端口映射

  • emule -> 选项 -> 连接，确认客户端口栏 TCP 和 UDP 的端口号，点测试端口，弹出的网页如果显示 test failed，说明需要端口映射
  • 先在自己的路由器上设置端口映射（或者叫虚拟服务器），外部端口和内部端口都写成刚才 emule 客户端查到的端口号，IP 地址写成运行 emule 电脑的局域网 IP。需要设置两条，分别对应 TCP 和 UDP 协议类型。其中 TCP 端口供 ed2K 服务器使用，UDP端口供 Kad 网络使用
  • 现在一般的上网猫会自带路由功能，所以在上网猫中也要按照上面的方法设置端口映射。这个操作需要管理员权限，通过上网猫型号网络搜索，一般可以查到管理员登录密码
  • 进入 emule，再次点测试端口，此时测试网页应该已经能显示 TCP/DUP test successful 了。重启 emule，得到 HighID
