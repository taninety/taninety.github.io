面试问题

list，set，map 有什么区别？

这些都是 java 的集合类

Set，继承自 Collection，表示一组对象的集合，元素无序且不重复

  • HashSet，散列形式，无法排序
  • TreeSet，以二叉树结构存储，可以对元素进行排序

List，继承自Collection，表示一组对象的集合，元素有序且允许重复

  • ArrayList，数组结构，线程不安全，对元素查询速度快
  • Vector，与ArrayList实现类似，线程安全
  • LinkedList，链表结构，对元素增删速度快

Map，元素成对出现，键值对

  • HashMap，线程不安全，允许存放null键和null值
  • HashTable，线程安全，不允许存放null键和null值
  • TreeMap，可以对键进行排序
  • LinkedHashMap，和HashMap相比，它是有序的

ArrayList 和 LinkedList 有什么区别？

都实现了 List 所有未实现的方法，但实现方式不同

  • ArrayList 于动态数组的数据结构，LinkedList基于链表的数据结构
  • 对于随机访问 get 和 set， ArrayList 优于 Linked
  • 对于新增和删除操作 add 和 remove，LinkedList 优于 ArrayList

ArrayMap 和 HashMap 有什么区别？

ArrayMap内部使用两个数组进行数据存储，一个数组记录key的hash值，一个数组记录Value值
HashMap内部使用一个默认容量为16的数组来存储数据，数组的每一个元素是是一个链表的头结点。

  • ArrayMap 内存占用小于 HashMap
  • ArrayMap 处理速度慢于 HashMap，ArrayMap 不适合大量数据

HashMap 和 Hashtable 有什么区别？

  • HashMap 继承于 AbstractMap，Hashtable 继承于 Dictionary
  • Hashtable 线程安全， HashMap 线程不安全
  • HashMap的key、value都可以为null， Hashtable的key、value都不可以为null
  • HashMap 只支持 Iterator 遍历，Hashtable 支持 Iterator(迭代) 和 Enumeration(枚举) 两种方式遍历
  • HashMap默认容量16，增加容量时原始容量x2，Hashtable默认容量大小11，增加容量时，原始容量x2+1
  • HashMap 添加元素时，使用自定义算法，Hashtable 直接采用 key的 hashCode()

HashMap 的实现原理？

HashMap内部使用一个默认容量为16的数组来存储数据，数组的每一个元素是是一个链表的头结点。

HashMap是线程安全的吗？

不是

如果HashMap特别大的话，什么情况下会触发扩容？

当 map 中包含的 Entry 数量大于等于 threshold = loadFactor * capacity 的时候，且新建的 Entry 正好落在一个非空的桶上，触发扩容机制，将容量扩大2倍
其中 loadFactor 默认为 0.75， capcity 默认为 16
